# Setup

## Requirements

- Node.js `^22.13.0 || >=24`
- pnpm `10.33.3`
- Expo tooling for device development
- EAS CLI for cloud builds

## Install

```bash
pnpm install
```

## Environment

The env schema is generated in `env.ts`. No `.env` file is required for default development.

Supported app environments:

- `development`
- `preview`
- `production`

Derived values:

- `EXPO_PUBLIC_NAME`
- `EXPO_PUBLIC_SCHEME`
- `EXPO_PUBLIC_BUNDLE_ID`
- `EXPO_PUBLIC_PACKAGE`
- `EXPO_PUBLIC_VERSION`

Strict validation is enabled by setting:

```bash
STRICT_ENV_VALIDATION=1
```

## Run The App

```bash
pnpm dev
pnpm android
pnpm ios
```

Environment helpers:

```bash
pnpm start:preview
pnpm start:production
pnpm android:preview
pnpm android:production
pnpm ios:preview
pnpm ios:production
```

## Quality Checks

```bash
pnpm verify
pnpm lint
pnpm lint:ts
pnpm lint:translations
pnpm test
pnpm test:watch
pnpm test:ci
pnpm expo:check
pnpm knip:check
```

`pnpm verify` runs Expo dependency checks, ESLint, TypeScript, translation lint, and Jest.

## Notes

- Use pnpm only; `preinstall` enforces it.
- `android/` and `ios/` are generated by Expo prebuild and are ignored.
- Expo Updates uses the committed EAS project ID; `EAS_PROJECT_ID` can override it locally.
- `pnpm e2e-test` expects Maestro and a development app id: `com.nejcm.dottodust.development`.
